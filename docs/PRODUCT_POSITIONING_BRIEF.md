# Product Positioning Brief

## Product Name
Antinode Norma

## Target Audience
- QA leaders who need faster regression coverage and lower automation maintenance costs.
- Automation engineers who want reliable BDD test generation with reusable page objects and step definitions.
- Product owners who need story-to-test traceability and faster validation of user-facing behavior.
- Engineering teams that want open-source, vendor-neutral AI automation tooling.

## Core Problem
Manual test creation is slow, error-prone, and expensive.
BDD adoption stalls because automation is brittle, step mappings are inconsistent, and generated tests break easily.
Teams need a predictable path from story to stable, executable tests that can be maintained alongside product changes.

## Key Benefits
- Generate stable Playwright automation directly from feature files.
- Preserve reusable test assets with page objects and shared step libraries.
- Reduce brittleness with hybrid LLM + rule-based mapping and failure-aware learning.
- Support open-source and self-hosted AI workflows, minimizing vendor lock-in.
- Keep QA aligned with development velocity through automated drift detection and repair guidance.

## Differentiators
- Open-source-first product with support for OpenRouter, Claude, and local model providers.
- Hybrid mapping engine that combines AI reasoning with rule-based safety and fallback.
- Story/feature-first workflow that produces structured test scripts, not one-off recorder code.
- Built-in diagnostics for mapping fidelity, selector robustness, and generated artifact quality.
- Roadmap centered on fleet engineering and lossless test asset reuse.

## Messaging Pillars
- “From story to stable tests in one workflow.”
- “AI-assisted step mapping, not AI-generated guesswork.”
- “Build reusable test assets, not brittle one-off scripts.”
- “Open-source automation with self-hosted AI compatibility.”
- “Keep QA aligned with product change through loop-aware automation.”

## Primary Use Cases
1. Generate Playwright tests from Gherkin feature files.
2. Validate and repair existing automation when selectors or step mappings drift.
3. Extend open-source test fleets with shared page objects and selector patterns.
4. Integrate generated tests into CI pipelines and issue tracking workflows.

## Proof Points
- Sample project with login, password reset, and checkout flows.
- Playwright validation workflow and GitHub Actions template.
- OpenRouter/self-hosted AI provider compatibility.
- Public roadmap and contributor-friendly documentation.
